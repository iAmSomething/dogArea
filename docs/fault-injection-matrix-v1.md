# Fault Injection Matrix v1 (P0/P1)

## 1. 목적
예외처리 검증을 기능 QA와 분리하여 반복 가능하게 운영하고, 릴리스 차단 기준을 명확히 한다.

## 2. 등급 정의
- `P0`: 사용자 데이터 유실/오동작/보안·인증 치명 이슈. 1건이라도 FAIL이면 릴리스 차단.
- `P1`: 복구 가능 이슈. 릴리스 전 우선순위 조정 대상이며 잔여 리스크를 명시한다.

## 3. 릴리스 게이트
- 최소 통과 기준
  - `P0`: 전 항목 `PASS`
  - `P1`: 핵심 경로(권한, 네트워크, 인증) `PASS`, 나머지는 우회/복구전략 문서화 시 조건부 허용
- 자동 블로킹 기준
  - `P0 FAIL >= 1`이면 `NO-GO`

## 4. 매트릭스
| ID | Priority | 영역 | 시나리오 | 기대 결과 | 로그 증거 |
|---|---|---|---|---|---|
| FI-P0-001 | P0 | 권한 | 산책 중 위치 권한 `Always/WhenInUse -> Denied` 강등 | 세션 안전 일시중지 + 복구 배너 + 데이터 유실 없음 | 앱 로그, UI 캡처 |
| FI-P0-002 | P0 | GPS | 저정확도/점프 포인트 혼입 | 이상 샘플 폐기, 면적 급등 없음 | runtime guard 로그 |
| FI-P0-003 | P0 | 네트워크 | 오프라인 저장 후 온라인 복귀 | 로컬 저장 성공 + outbox 순차 재전송(`session->points->meta`) | outbox 상태 로그 |
| FI-P0-004 | P0 | 인증 | 토큰 만료(401/403) 후 재인증 | 큐 보존 + 재인증 후 재동기화 재개 | API 응답/재시도 로그 |
| FI-P0-005 | P0 | 시간 경계 | 자정 경계/주간 경계에서 산책 종료 | 통계/집계 경계 일관성 유지 | 통계 결과 스냅샷 |
| FI-P1-001 | P1 | 저장공간 | 이미지 저장 실패(디스크/권한) | 세션/포인트 저장은 성공, 이미지 실패만 분리 표기 | 사용자 메시지 + 저장 로그 |
| FI-P1-002 | P1 | 워치 연결 | 워치 비연결 상태 액션 누적 후 재연결 | 로컬 큐 누적 후 중복 없이 재전송 | watch action 로그 |
| FI-P1-003 | P1 | 서버 과부하 | 5xx/429 응답 | backoff 재시도, 앱 크래시 없음 | 재시도 횟수 로그 |

## 5. 운영 규칙
- 예외 매트릭스 결과는 릴리스 PR마다 링크를 남긴다.
- 결과는 텍스트 요약이 아니라 재현 절차 + 로그 링크를 포함한다.
- 자동화 불가 케이스는 수동 테스트로 명시하고 담당자/실행일을 기록한다.
