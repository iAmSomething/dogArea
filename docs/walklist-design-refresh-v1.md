# WalkList Design Refresh v1

## 목적
- 산책 목록 화면을 `최근 산책 기록을 탐색하고 다음 행동을 결정하는 허브`로 재정의합니다.
- 기존 목록 필터 정책과 상세 라우팅은 유지하면서, 상단 정보 구조와 셀 스캔 속도를 개선합니다.

## 상단 허브 구조
1. 화면 제목 + 보조 설명
2. 최근 요약 카드
   - 총 기록 또는 선택 기준 기록 수
   - 이번 주 기록 수
   - 현재 보이는 기준의 누적 영역
3. 필터 문맥 카드
   - `선택 반려견 기준` / `전체 기록 보기` / `전체 반려견 기준` badge
   - 현재 왜 이 기준으로 보고 있는지 설명
   - 반려견 칩
   - 필요 시 `기준으로 돌아가기` CTA

## 상태 카드 원칙
- `게스트 모드`
  - 로그인 CTA 유지
  - 기록은 로컬에만 저장된다는 점을 명시
- `기록 없음`
  - 지도 탭에서 산책을 시작하라는 다음 행동을 명확히 안내
- `선택 반려견 필터 결과 0건`
  - 왜 0건인지 설명
  - `전체 기록 보기` CTA 제공

## 리스트 셀 구조
- 날짜 / 시작 시각
- 반려견 배지
- 영역 중심 헤드라인
- 산책 시간 / 영역 넓이 / 포인트 수 / 반려견 4개 메트릭 타일
- 썸네일 또는 기록 요약 placeholder
- chevron affordance 유지

## 시각 원칙
- 얇은 border 반복 대신 라운드 surface + 내부 metric tile rhythm 사용
- 지도/설정과 같은 spacing, typography, background tone 사용
- Dynamic Type 시 카드 높이는 자연스럽게 증가하고 텍스트가 잘리지 않아야 함

## 유지해야 하는 기존 동작
- `WalkListDetailView` 네비게이션
- pull to refresh
- 게스트 로그인 CTA
- 선택 반려견 필터 / 전체 기록 보기 전환
