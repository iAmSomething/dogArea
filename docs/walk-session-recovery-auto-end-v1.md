# Walk Session Recovery & Auto-End v1

## 1. 목적
앱 재실행/백그라운드 전환/종료 누락 상황에서 산책 기록 유실을 줄이기 위해 세션 복구와 자동 종료 정책을 도입한다.

## 2. 세션 임시저장
- 저장 키: `active_walk_session` (`UserDefaults`)
- 저장 시점
  - 산책 시작 직후
  - 타이머 tick(1초) 중 주기 저장
  - 위치 업데이트 수신 시
  - 앱 `willResignActive`, `willTerminate`
- 저장 내용
  - 시작 시각, 경과 시간, 포인트 배열, 선택 반려견, 포인트 기록 모드

## 3. 재실행 복구 UX
- 앱 시작 시 `active_walk_session` 존재 여부 확인
- 복구 후보 조건
  - 저장 시각 기준 12시간 이내
  - 경과 시간 또는 포인트가 존재
- 재실행 분기
  - 자동 확정/자동 종료 금지: 항상 사용자 확인 후 확정
  - 지도 상단 배너에서 `복구 / 추정 종료 / 지금 종료 / 닫기` 제공
  - `추정 종료`는 `lastMovementAt + 15분` 기준 시각을 제안하며 사용자가 수동 확정
  - 추정 종료 실패 시 draft 보존 + 실패 토스트 + 메트릭 로그 적재

## 4. 자동 종료 정책(v1)
- 설정: `자동 종료 정책 v1` 고정(비활성화 불가)
- 설정 화면 노출: 고정 정책 태그 + 단계/판정 기준 안내 문구 표시
- 무이동 유효 샘플 조건
  - `horizontalAccuracy <= 40m`
  - `speed < 0.3m/s` 이고 앵커 대비 `distance < 25m`이면 무이동으로 판정
- 단계 정책
  - `5분` 무이동: 휴식 후보 알림
  - `12분` 무이동: 자동 종료 예정 알림
  - `15분` 무이동: 자동 종료 확정(`reason=auto_inactive`)
- 동작
  - 포인트 3개 이상: 자동 저장 종료
  - 포인트 3개 미만: 임시 기록 폐기
- 종료 시각 보정: `endedAt = min(now, lastMovementAt + 15분)`
- 최대 산책 시간(`1시간`) 초과 시 `reason=auto_timeout`으로 종료
- 사용자 메시지: 자동 종료/폐기 결과를 토스트로 노출

## 5. Watch 반영 상태
- 워치 컨텍스트 동기화 성공 시각 표시
- 워치 액션 수신/반영 이벤트를 상태 문구로 노출

## 6. QA 체크 포인트
- 강제종료 후 재실행 시 복구 배너 노출
- `복구` 선택 시 이전 시간/포인트 상태 재개
- `추정 종료` 선택 시 제안 종료 시각으로 수동 확정 저장
- `지금 종료` 선택 시 현재 시각 기준 수동 저장
- 종료 확정 실패 시 draft가 보존되어 재시도 가능
- 5/12/15분 단계 정책(휴식 후보/경고/자동 종료) 동작
- 무이동 유효 샘플(accuracy/speed/distance) 조건 검증
- 워치 액션 수신 후 상태 문구 업데이트 확인
