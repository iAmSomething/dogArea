# Fault Injection Runbook v1

## 1. 범위
- 대상: `docs/fault-injection-matrix-v1.md`의 P0/P1 시나리오
- 실행 환경: Debug 빌드, 테스트 계정, Supabase 연결 환경

## 2. 공통 기록 포맷
- 각 시나리오마다 다음 4가지를 반드시 기록:
  - 재현 절차
  - 기대값
  - 실제값
  - 로그 링크/스크린샷 경로

## 3. 시나리오별 장애주입 절차

### 3.1 토큰 만료 (FI-P0-004)
1. 로그인 후 산책 기록을 생성한다.
2. `SUPABASE_ANON_KEY`를 만료/무효 토큰으로 교체하거나 서버에서 401/403을 강제한다.
3. 동기화 요청을 발생시킨다.
4. 기대값:
   - outbox 항목이 보존된다.
   - 오류코드가 `token_expired`로 기록된다.
   - 재인증 후 flush 재개 시 성공으로 수렴한다.

### 3.2 네트워크 단절 (FI-P0-003)
1. 기기 네트워크를 끈 상태에서 산책 종료 저장을 실행한다.
2. outbox 대기 건수를 확인한다.
3. 네트워크 복구 후 자동/수동 flush를 실행한다.
4. 기대값:
   - 로컬 저장 성공
   - `session -> points -> meta` 순서 재전송
   - 최종 대기 건수 0 또는 재시도 상태로 합리적 수렴

### 3.3 위치 튐/저정확도 (FI-P0-002)
1. 테스트 좌표에 저정확도(`horizontalAccuracy` 큰 값) 샘플을 주입한다.
2. 단시간 장거리 점프 좌표를 섞어 주입한다.
3. 기대값:
   - 저정확도/점프 포인트는 폐기
   - 면적/경로가 비정상 급등하지 않음

### 3.4 권한 강등 (FI-P0-001)
1. 산책 중 설정 앱에서 위치 권한을 `Denied`로 변경한다.
2. 앱으로 복귀한다.
3. 기대값:
   - 세션 안전 일시중지
   - 복구 배너 노출
   - 기존 포인트 유실 없음

### 3.5 저장공간/이미지 실패 분리 (FI-P1-001)
1. 이미지 캡처 실패 상황을 만들고 산책 종료를 진행한다.
2. 기대값:
   - 세션/포인트 저장 성공
   - 이미지 실패만 메시지/메타(`has_image=false`)로 분리

## 4. 결과 업로드 규칙
- 결과는 이슈 코멘트 또는 릴리스 PR에 링크한다.
- 파일명 규칙: `fault-injection-<날짜>-<시나리오ID>.md`
- 실패 케이스는 `원인/재현성/차단여부(P0/P1)`를 함께 남긴다.
