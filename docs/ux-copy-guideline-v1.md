# UX Copy Guideline v1

## 목적
- 사용자 화면에서 개발자/시스템 중심 용어가 직접 노출되지 않도록 기준을 고정한다.
- 버튼, 배지, 토스트, 설정 항목, 상태 메시지 전반에 같은 용어 원칙을 적용한다.
- 이후 화면 작업과 QA에서 바로 복붙해 쓸 수 있는 대표 치환 예시를 제공한다.

## 기본 원칙
- 기능 구현 방식보다 사용자가 얻게 되는 결과를 먼저 쓴다.
- 내부 상태명, fallback 분기, 실험 용어, DB/RPC 개념은 그대로 노출하지 않는다.
- 한 번 읽고 바로 행동을 알 수 있는 짧은 문장으로 쓴다.
- 같은 기능은 화면이 달라도 핵심 용어를 통일한다.
- 숫자/상태만 보여주지 말고, 필요할 때는 짧은 의미 문구를 함께 둔다.

## 금지/지양 용어
| 내부 용어 | 원칙 | 사용자 노출 대체 방향 |
| --- | --- | --- |
| `Fallback` | 금지 | `날씨 정보를 잠시 불러오지 못했어요`처럼 결과 중심으로 설명 |
| `Heatmap` | 지양 | `시즌 점령 지도`, `자주 걷는 구역`, `활동 분포`처럼 의미 중심 용어 사용 |
| `Shield` | 지양 | `보호`, `유지`, `지켜줌`처럼 결과를 한국어로 설명 |
| `AUTO` | 금지 | `자동 기록`, `자동 추가`처럼 동작을 드러내는 문구 사용 |
| `Undo` | 금지 | `실행 취소` 사용 |
| `today` | 금지 | `오늘`, `오늘 기준` 사용 |
| `Fallback mode`, `debug`, `experimental`, `legacy` | 금지 | 개발/운영 내부 맥락은 사용자에게 노출하지 않음 |

## 영어 사용 기준
- 시스템/플랫폼 고유명사는 유지 가능: `iOS`, `GPS`, `Apple Watch`, `TestFlight`.
- 브랜드/외부 서비스명은 유지 가능: `OpenAI`, `Supabase`.
- 기능 라벨, 버튼, 토스트, 배지에서는 불필요한 영어를 쓰지 않는다.
- 같은 의미를 한국어로 자연스럽게 전달할 수 있으면 한국어를 우선한다.

## 요소별 작성 원칙

### 버튼
- 동사로 시작하거나, 탭 이후 결과가 바로 떠오르게 쓴다.
- 구현 방식이 아니라 사용자 행동을 기준으로 쓴다.
- 예: `위치 다시 맞추기`, `자동 기록 켜기`, `실행 취소`

### 배지
- 한 단어 또는 짧은 두 단어로 끝낸다.
- 상태명만 던지지 말고, 의미가 불명확하면 보조 문구를 카드/토스트로 분리한다.
- 예: `자동 기록`, `보호 중`, `오늘 +3`

### 토스트/상태 메시지
- “무엇이 되었는지 + 사용자가 지금 이해해야 할 다음 문맥” 순으로 쓴다.
- 오류는 내부 실패 원인이 아니라 사용자가 체감한 문제를 설명한다.
- 예: `날씨 정보를 잠시 불러오지 못했어요. 기본 화면으로 보여드릴게요.`

### 설정 항목
- 토글 기술명을 쓰지 말고 켰을 때의 결과를 설명한다.
- 예: `시즌 점령 지도 표시`가 `Heatmap`보다 우선
- 예: `영역 추가 후 바로 되돌리기 허용`보다 `영역 추가 후 실행 취소 허용` 사용

### 카드/요약 수치
- 숫자만 강조하지 말고 기간/의미를 함께 쓴다.
- 예: `+16 today` 대신 `오늘 +16점`
- 예: `Shield 1회` 대신 `보호 1회`

## 대표 치환 예시
| 변경 전 | 변경 후 |
| --- | --- |
| `Fallback: 날씨 데이터 연결 불가` | `날씨 정보를 잠시 불러오지 못했어요` |
| `Heatmap` | `시즌 점령 지도` |
| `AUTO` | `자동 기록` |
| `영역 추가: 1탭+Undo` | `영역 추가 후 실행 취소 가능` |
| `Shield` | `보호` |
| `+0 today` | `오늘 +0` |
| `Fallback 영역` | `기본 비교 구역` |
| `최근 uplink 실패` | `최근 동기화가 늦어지고 있어요` |

## 화면 적용 우선순위
1. 지도 상단 상태/배지/토스트
2. 지도 설정 시트
3. 홈 시즌/날씨/목표 요약 카드
4. 라이벌/위젯의 짧은 상태 배지

## QA 체크포인트
- 문구만 읽고도 사용자가 버튼 결과를 유추할 수 있어야 한다.
- 영어를 제거한 뒤에도 의미 손실이 없어야 한다.
- 같은 기능을 다른 화면에서 부를 때 핵심 용어가 달라지지 않아야 한다.
- 서버/개발자 내부 상태명을 그대로 보여주는 문구가 남아 있지 않아야 한다.

## 후속 적용 범위
- 지도/홈 1차 sweep: `#461`
- 이후 신규 화면/리팩터링 PR에서도 본 문서를 기본 가이드로 사용한다.
