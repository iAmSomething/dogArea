# Walk Start/Stop UX v1

## 1. 목적
산책 시작/종료 플로우의 강제 대기를 제거하고, 사용자가 즉시 제어 가능한 단순한 UX로 정리한다.

## 2. 변경 요약
- 기본 동작을 `즉시 시작`으로 변경한다.
- 설정에서 `시작 카운트다운(3초)`를 선택적으로 켜고 끌 수 있다.
- 카운트다운 화면에 `취소` 버튼을 제공한다.
- 산책 종료 의사결정을 3가지로 명확히 분리한다.
  - `저장하고 종료`
  - `계속 걷기`
  - `기록 폐기`

## 3. 시작 플로우
### 3.1 기본(카운트다운 OFF)
1. 시작 버튼 탭
2. 즉시 산책 시작

### 3.2 선택형(카운트다운 ON)
1. 시작 버튼 탭
2. 3초 카운트다운 표시
3. 사용자 선택
   - 완료: 산책 시작
   - 취소: 시작 취소

## 4. 종료 플로우
1. 종료 버튼 탭
2. 3액션 알림 표시
   - `저장하고 종료`: 기존 상세 저장 흐름 진입
   - `계속 걷기`: 아무 상태 변경 없음
   - `기록 폐기`: 현재 세션 포인트/시간 폐기 후 미산책 상태로 복귀

## 5. Watch 정합성
- Watch `startWalk` 액션은 강제 대기 없이 즉시 시작 정책을 따른다.
- 시작/종료 후 Watch context(`isWalking`, `time`, `area`, `last_sync_at`)를 즉시 동기화한다.

## 6. 회귀 체크 포인트
- 카운트다운 OFF에서 시작 탭 즉시 기록 시작
- 카운트다운 ON에서 취소 시 세션 미시작
- 종료 3액션 각각의 결과가 의도대로 반영
- 산책 중/종료 후 Watch 상태 동기화 정상
