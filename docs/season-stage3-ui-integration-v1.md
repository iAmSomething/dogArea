# Season Stage 3 UI Integration v1

## 목표
- 시즌 상태를 Home/Map/Profile에 일관되게 노출한다.
- 시즌 종료 결과와 보상 수령 실패 재시도 흐름을 완성한다.
- 지도는 점수의 정밀값 대신 4단계 강도와 점령/유지 상태를 제공한다.

## 화면별 계약

### 1) Home
- 시즌 카드는 `티어/절대 점수/오늘 증가분/남은 시간`을 표시한다.
- 시즌 카드는 `접기/펼치기`를 지원한다.
- 시즌 상세 시트에서 점수 구성(기여/Shield/랭크 구간)을 확인한다.
- 시즌 종료 결과는 자동 노출 후 재확인 가능하다.
- 보상 상태(`pending/claimed/failed`)를 결과 모달에서 노출한다.
- 보상 수령 실패 시 재수령 버튼을 제공한다.

### 2) Map
- Heatmap 셀은 `4단계 강도`로 렌더링한다.
- 각 셀은 점수 구간으로 `점령/유지` 상태를 분류한다.
- 지도 상단/설정에서 시즌 타일 상태 요약(점령 n, 유지 n)과 범례를 제공한다.

### 3) Profile
- 시즌 티어 기반 배지/프레임을 프로필 이미지에 반영한다.
- 시즌 요약 카드와 이미지 프레임의 티어가 동일 규칙을 사용한다.

## 상태 모델
- `SeasonMotionSummary`
  - `score`: 시즌 누적 점수(절대값)
  - `todayScoreDelta`: 오늘 증가분
  - `rankTier`, `contributionCount`, `weatherShieldApplyCount`, `weekKey`
- `SeasonRewardClaimStatus`
  - `pending | claimed | failed`

## 성능/안정성
- 지도 오버레이는 기존 LOD/클러스터링 제약을 유지한다.
- 시즌 타일 UI는 색상/요약 계산만 추가하고, 도형 수 증가를 유발하지 않는다.

## QA 체크포인트
1. 시즌 점수 증가 시 Home의 절대값/오늘 증가분이 동시에 반영된다.
2. 시즌 종료 결과 모달에서 보상 상태와 재수령 버튼 동작이 일치한다.
3. Map에서 heatmap 강도가 4단계로 표시되고 점령/유지 카운트가 갱신된다.
4. Profile 이미지 프레임/배지가 시즌 티어와 일치한다.
