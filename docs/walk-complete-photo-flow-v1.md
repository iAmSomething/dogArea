# Walk Complete Photo Flow v1

## 목적
- 산책 완료 화면에서 사용자가 사진을 촬영하고, 산책 요약 정보(날짜/시간/넓이/포인트)를 포함한 카드로 저장/공유한다.

## 대상 화면
- `WalkDetailView` (산책 종료 직후)

## 핵심 시나리오
1. 사용자가 `사진 찍기`를 누른다.
2. 카메라 촬영 결과를 상단 프리뷰에 반영한다.
3. `저장하기`는 원본이 아닌 정보 오버레이 카드(1080x1080)를 사진 앱에 저장한다.
4. `공유 시트 열기`도 동일 카드 이미지를 첨부한다.

## 데이터 규격
- 카드 포함 정보:
  - 산책 날짜/시간
  - 산책 시간
  - 영역 넓이(㎡ + 평)
  - 영역 포인트 수
  - 반려견 이름
- 이미지 소스 우선순위:
  - 촬영 이미지
  - 지도 스냅샷
  - 둘 다 없으면 텍스트-only fallback

## 예외 처리
- 카메라 미지원 기기(시뮬레이터 등)에서는 사진 라이브러리 선택으로 fallback
- 촬영 취소 시 기존 상태 유지

## QA 체크
- [ ] `사진 찍기` 탭 시 카메라가 열리고, 완료 후 프리뷰가 교체된다.
- [ ] 카메라 미지원 환경에서 라이브러리 fallback이 열린다.
- [ ] `저장하기` 결과 이미지에 날짜/시간/넓이/포인트가 포함된다.
- [ ] `공유 시트 열기` 결과 이미지에 동일 정보가 포함된다.
