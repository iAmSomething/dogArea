# 지도 퀘스트 Companion HUD 최소 정보셋 v1

## 목적
산책 중 지도 HUD는 `1~2초 안에 이해`되는 수준으로만 정보를 남겨야 합니다. 이 문서는 `#465`가 정의한 HUD/토스트/체크리스트 계층 중, Companion HUD 자체에 무엇을 남기고 무엇을 숨길지 고정합니다.

## 기본 원칙
- collapsed HUD는 `2줄 + 상태 배지 1개`를 넘기지 않습니다.
- 기본 HUD는 `full-width progress bar`를 쓰지 않습니다.
- 숫자는 `진행률 퍼센트`보다 `남은 조건 한 줄`을 우선합니다.
- 대표 미션이 없으면 empty state 문구만 남기고, 강한 CTA는 두지 않습니다.
- 다중 미션이어도 `대표 1개 + 추가 n개` 규칙을 유지합니다.

## collapsed HUD 최소 정보셋

### 공통 구조
- 1행: 대표 미션명
- 2행: `남은 시간 8분`, `남은 타일 1칸`, `200m만 더 걸으면 끝나요`처럼 남은 조건 1줄
- 보조 배지: 아래 1개만 허용
  - `거의 완료`
  - `완료`
  - `보상 가능`
  - `+n개 진행 중`

### 길이 제한
- 제목: 최대 `16자`
- 요약: 최대 `22자`
- 줄 수: 최대 `2줄`
- 넘치면 뒤에 `...` 처리

### 숫자 표현 규칙
- 기본: `남은 2회`, `남은 시간 8분`, `남은 거리 200m`
- 완료/보상 가능 상태만 예외적으로 `완료`, `보상 가능` 배지 우선
- `67%`, `3/5` 같은 비율형 표현은 expanded 상태에서만 보조적으로 허용
- collapsed HUD에서는 `progress bar`를 숨깁니다

## 상태별 축약 규칙

### 1. 미션 1개 진행
- 제목: 대표 미션명
- 요약: 남은 조건 1줄
- 배지: 없음
- 예시
  - `산책 20분 채우기`
  - `남은 시간 8분`

### 2. 다중 미션 동시 진행
- 제목: 대표 미션 1개만 유지
- 요약: 대표 미션의 남은 조건 1줄
- 배지: `+2개 진행 중`처럼 추가 개수만 표시
- 다른 미션 제목은 collapsed HUD에 올리지 않음
- 예시
  - `새 영역 3칸 확보`
  - `남은 타일 1칸`
  - badge: `+2개 진행 중`

### 3. 완료 직전
- 제목: 대표 미션명 유지
- 요약: 마지막 조건만 직접적으로 표현
- 배지: `거의 완료`
- 예시
  - `거리 미션 거의 완료`
  - `200m만 더 걸으면 끝나요`

### 4. 완료 후
- 제목: `퀘스트 완료`
- 요약: `이번 산책 조건을 모두 채웠어요`
- 배지: `완료`
- auto-collapse: `6초` 후 compact 또는 기본 상태로 복귀

### 5. 보상 가능
- 제목: `보상 받을 수 있어요`
- 요약: `홈 퀘스트 보드에서 이어서 마무리해요`
- 배지: `보상 가능`
- auto-collapse: `8초` 후 compact 또는 기본 상태로 복귀

### 6. 대표 미션 없음
- 제목: `진행 중인 퀘스트 없음`
- 요약: `산책을 시작하면 자동 기록 미션을 여기서 보여줘요`
- 배지: 없음

## expanded 상태에서만 보여줄 정보
collapsed HUD에 넣지 않고, 사용자가 펼쳤을 때만 보여줄 정보는 아래입니다.
- 체크리스트 전체 항목
- 보상 설명
- 세부 조건 breakdown
- 진행률 bar / 비율형 수치
- 홈 전용 직접 체크 항목 설명

정리하면:
- collapsed는 `대표 미션 1개를 이해시키는 층`
- expanded는 `왜 그런지 설명하는 층`

## 작은 화면 / 다중 배너 기준
- HUD 높이는 `72pt`를 넘기지 않습니다.
- critical banner가 있으면 HUD는 숨기거나 compact 상태로 밀립니다.
- 작은 화면에서 배지와 텍스트가 동시에 길어지면 `배지 우선`, 제목/요약은 더 강하게 축약합니다.
- 지도 pan/zoom을 막지 않도록 hit area는 HUD 카드 내부로 제한합니다.

## 지도 조작을 방해하지 않는 최소 시각 무게
- collapsed HUD는 배경 강조보다 `짧은 카피 + 상태 배지`로 의미를 전달합니다.
- 강한 변화는 HUD 확대가 아니라 `milestone toast`가 담당합니다.
- 완료/보상 가능 상태에서도 카드 자체 높이나 광원 효과를 키우지 않습니다.
- 대표 미션이 없을 때는 empty state를 작고 중립적으로 유지합니다.

## Widget / Home 정렬 메모
- widget은 같은 상태를 더 짧게 축약해 mirror 합니다.
- home은 expanded 설명과 claim context를 담당합니다.
- map HUD는 `상황을 알리는 층`이지, 모든 퀘스트 정보를 다 보여주는 보드가 아닙니다.
