# Map Season Tile Detail Panel v1

## 목표
- 지도에서 시즌 타일을 눌렀을 때 `왜 이 칸이 이런 상태인지`와 `다음 산책에서 무엇을 하면 좋은지`를 바로 이해할 수 있게 한다.
- 시즌 점령 지도는 전체 요약만이 아니라 공간 단위 drill-down을 제공한다.

## 상호작용 결정
- 선택 방식: `타일 중심 hit target 탭`
- 상세 표면: `상단 floating detail card`
- 접근성 fallback: `요약 카드의 대표 칸 상세 보기 버튼`
- 닫기 방식: `카드 우측 상단 닫기 버튼` 또는 같은 타일 재탭
- walking 중 처리: 별도 축약 패널을 만들지 않고, 기존 `season tile map hidden while walking` 규칙을 그대로 따른다.

## 왜 bottom sheet가 아닌가
- 지도 하단에는 산책 CTA, 영역 추가, 선택 트레이가 이미 존재한다.
- 이 이슈의 목적은 탐색 흐름을 끊지 않는 경량 해설이므로, 하단 sheet는 정보 밀도보다 충돌 비용이 더 크다.
- 상단 chrome 아래 카드가 지도 탐색과 하단 CTA 우선순위를 가장 덜 침해한다.

## 패널에 보여주는 정보
- 현재 상태: `점령 / 유지`
- 강도: `1단계 ~ 4단계`
- 상태 이유: 현재 타일이 그렇게 보이는 이유를 한 문장으로 해설
- 다음 행동: 다음 산책에서 취하면 좋은 행동 힌트
- 현재 기여 요약: 강도와 기여 정도를 짧게 설명하는 문장

## 사용자 문구 원칙
- 디버그 값, geohash, 내부 점수 원문은 노출하지 않는다.
- 문구는 `지금 상태의 의미`와 `다음 행동` 중심으로 작성한다.
- 홈의 영역 목표 설명과 개념이 충돌하지 않도록 `시즌 칸`, `점령`, `유지`, `다음 산책 힌트` 표현을 사용한다.

## 시각 우선순위
- 선택된 타일은 노란 테두리로 강조한다.
- 상세 카드가 열려도 지도는 그대로 유지하되, 정보는 상단 카드 하나로 제한한다.
- 하단 CTA와 탭바는 기존 우선순위를 유지한다.

## 성능 원칙
- 상세 패널은 기존 `seasonTileMapTiles`를 재사용한다.
- 선택/해제는 `selectedSeasonTileGeohash`만 변경하고, 서버 호출이나 타일 재집계를 유발하지 않는다.
- 패널 문구는 `MapSeasonTilePresentationService`가 기존 타일 표현 모델을 기반으로 생성한다.

## QA 체크포인트
1. 시즌 타일을 탭하면 상세 패널이 열린다.
2. 타일 hit target이 없는 환경에서도 `대표 칸 상세 보기` fallback으로 동일 상세 패널을 열 수 있다.
3. 패널에 상태 이유와 다음 행동 힌트가 모두 노출된다.
4. 같은 타일을 다시 누르거나 닫기 버튼을 누르면 패널이 닫힌다.
5. 선택된 타일은 비선택 타일과 구분되는 시각 강조를 가진다.
6. 패널이 열려도 하단 산책 CTA와 탭바를 가리지 않는다.
