# Territory Goal Detail UI v1

## 1. 목적
`TerritoryGoalView`를 홈의 요약형 `영역 목표 트래커`와 역할을 분리해, "왜 이 목표를 향해 가야 하는지"와 "다음 산책에서 무엇을 하면 되는지"를 더 자세히 전달하는 상세 화면으로 설계한다.

연결 컨텍스트:
- 홈 요약 카드: `HomeGoalTrackerCardView`
- 상세 화면: `TerritoryGoalView`

## 2. 홈 요약 vs 상세 화면 차별화 원칙
| 구분 | 홈(요약) | TerritoryGoalView(상세) |
| --- | --- | --- |
| 목적 | 빠른 상태 확인 + 진입 유도 | 의사결정 지원(다음 행동/우선순위) |
| 정보 밀도 | 핵심 수치 3개 중심 | 비교군/최근 정복/목표 달성 힌트까지 확장 |
| 체류 시간 | 짧음(스캔형) | 중간(탐색형) |
| 액션 | `목표 상세 보기` 1개 | 비교군 이동, 최근 정복 탐색, 새로고침 |
| 상태 처리 | 정상 상태 우선 | empty/offline/stale 상태 명시 |

## 3. 화면 정보 구조(상세 전용)
1. 컨텍스트 헤더
- 선택 반려견 기준 배지
- 현재 목표의 의미 문장(예: "다음 랜드마크까지 0.10km²")
- 데이터 기준 출처(Feature/Fallback) 노출

2. 목표 개요 카드(상세형)
- 현재 영역
- 다음 목표 이름/면적
- 남은 면적 + 진행률(%)
- 진행률 설명 문구(행동 유도형)

3. 인사이트 요약 2칸
- 최근 정복 개수
- 다음 목표까지 남은 면적
- 홈에는 없는 "최근 추세" 맥락을 숫자로 강조

4. 최근 정복 영역 리스트
- 최신순 3개 이상
- 획득 시각/증가 면적
- 최신 항목 `NEW` 배지

5. 빈 상태 행동 카드
- "산책으로 영역 확장" 안내
- 기능 미사용 유저를 위한 온보딩 문구

## 4. 상세 화면에 추가할 정보(기획 우선순위)
### P1 (현재 데이터로 즉시 가능)
- 최근 정복 상위 N개(현재 3개 -> 5개 확장 검토)
- 목표 대비 부족 면적을 거리/시간 힌트로 환산해 안내 문구 강화
- 마지막 갱신 시각(데이터 신선도 표시)

### P2 (서비스/도메인 확장 필요)
- 일/주 증가량 추세(미니 스파크라인)
- 목표 달성 예상 산책 횟수(최근 평균 기반)
- 목표 변경 이력(이전 목표 대비 난이도)

## 5. 인터랙션 정책
1. 홈 -> 상세 진입 시 하단 탭바 숨김 유지.
2. 상세 화면의 `비교군 카탈로그` CTA는 `AreaDetailView`로 이동.
3. pull-to-refresh 시 영역 목록 + 기준 카탈로그를 동시에 갱신.
4. 최근 정복 행 탭 시(차기): 해당 영역 상세로 drill-down.

## 6. 상태 설계(상세 차별 포인트)
1. `ready`: 전체 섹션 노출.
2. `empty`: 목표 카드 최소 정보 + 온보딩 카드 강조.
3. `offline`: 마지막 성공 스냅샷 유지 + 오프라인 배지.
4. `error`: 재시도 버튼과 원인 메시지 노출.

## 7. 접근성/테마 정책
1. 터치 타깃 최소 44pt 유지.
2. Dynamic Type에서 카드 높이 고정 금지.
3. 라이트/다크 모두 명도 대비 4.5:1 이상 확보.
4. 진행률은 색상+텍스트 동시 제공.

## 8. MVVM/파일 분리 제안(1 파일 1 뷰 기준)
`TerritoryGoalView` 비대화 방지를 위해 하위 뷰를 분리한다.

- `TerritoryGoalView.swift`: 화면 조립, 네비게이션, refresh 트리거만 담당
- `TerritoryGoalHeaderSectionView.swift`
- `TerritoryGoalOverviewCardView.swift`
- `TerritoryGoalInsightSectionView.swift`
- `TerritoryGoalRecentListSectionView.swift`
- `TerritoryGoalActionHintCardView.swift`

ViewModel 분리 기준:
- 단순 표시 계산은 `TerritoryGoalViewModel` 유지
- 독립 API/독립 상태가 생기면 하위 ViewModel 분리

## 9. 구현 완료 기준(기획 관점)
1. 홈 요약 카드와 정보 중복률 50% 이하(상세 정보 우위).
2. 사용자 질문 "다음에 뭘 하면 되는지"를 화면 내 텍스트만으로 설명 가능.
3. 상세 화면 진입 후 3초 내 핵심 지표(현재/다음/남은 면적) 스캔 가능.
