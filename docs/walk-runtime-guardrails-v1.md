# Walk Runtime Guardrails v1 (P0)

## 1. 목적
산책 중 런타임 상태 꼬임(위치 튐/권한 강등/중복 액션/타이머 드리프트)을 방어해 기록 유실과 오탐을 줄인다.

## 2. 위치 샘플 유효성
- 정확도 필터: `horizontalAccuracy > 0 && <= 65m` 샘플만 산책 계산에 반영
- 점프 필터:
  - 추정 속도 `> 12 m/s` 이면서 비정상 이동이면 폐기
  - `10초` 이내 `150m` 이상 점프 샘플 폐기
- 폐기 샘플은 면적/자동종료/자동포인트 계산에서 제외

## 3. 권한 강등 안전 일시중지
- 산책 중 권한이 `authorized -> denied/restricted`로 바뀌면 세션을 즉시 일시중지
- 임시 스냅샷 유지 후 사용자에게 `복구/폐기` 선택 제공
- 조용한 폐기는 금지

## 4. 액션 멱등 처리
- Watch 액션은 `action_id` 기반 중복 제거 유지
- `action_id` 누락 시 `action + sent_at`로 안정적인 대체 키 생성
- 상태 불일치 액션 수신 시 무시하지 않고 즉시 상태 재동기화

## 5. 타이머 드리프트 보정
- 누적 tick 기반이 아니라 `wall-clock(startTime 기반)`으로 경과시간 계산
- 백그라운드 복귀 후 실제 경과시간과 표시시간이 수렴하도록 보정

## 6. 앱/워치 재동기화
- 워치가 상태 재동기화 요청(`syncState`)을 보내면 iPhone이 즉시 context 재전송
- start/add/end가 현재 iPhone 상태와 맞지 않아도 context를 강제 동기화

## 7. QA 체크 포인트
- 저정확도 샘플 연속 입력 시 면적 급증 오탐 없음
- 동일 액션 중복 수신 시 세션 상태 일관성 유지
- 권한 강등 즉시 일시중지 + 복구 배너 제공
- 장시간 백그라운드 복귀 후 시간 드리프트 최소화
- 워치 `syncState` 요청 시 1초 이내 상태 재동기화
