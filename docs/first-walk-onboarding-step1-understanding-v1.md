# First Walk Onboarding Step1 Understanding v1

- Issue: #703
- Relates to: #271, #266

## 목표
- 첫 산책 전에 사용자가 `무엇이 기록되고`, `그 기록이 어디로 이어지며`, `미션은 어디에 놓이는지`를 10초 안에 이해하게 만든다.
- 설정 선택 UI보다 먼저 `산책의 의미`를 잡아주는 집중 설명 단계로 사용한다.

## 사용자가 Step1에서 답을 얻어야 하는 질문
- 산책을 시작하면 무엇이 기록되는가
- 이 기록이 어디에 이어지는가
- 실내 미션은 기본 루프의 어느 위치에 놓이는가

## 정보 우선순위
1. 기록
2. 영역
3. 시즌
4. 미션

Step1은 위 순서를 반드시 지킨다.

## 카피 원칙
- 문장은 1줄 또는 2줄 안에서 끝낸다.
- 하나의 카드에는 하나의 메시지만 둔다.
- 설명형 문단 대신 `제목 + 짧은 설명 + 결과 연결` 구조를 쓴다.
- Step1에서는 설정값, 권한 요청, 공유 ON/OFF 선택을 먼저 보여주지 않는다.

## 화면 구조

### 헤더
- 제목: `첫 산책 전에 이것만 보면 돼요`
- 보조 문구: `산책 기록이 어디로 이어지는지 짧게 알려드릴게요.`

### 핵심 카드 4장

#### 카드 1. 기록
- 제목: `산책을 시작하면 경로와 시간이 기록돼요`
- 설명: `이동 경로, 산책 시간, 포인트가 \(selectedPetName) 기준으로 쌓여요.`
- 사용자 인식 목표: “걷는 행동이 그냥 사라지지 않고 저장된다.”

#### 카드 2. 영역
- 제목: `기록은 우리집 영역으로 이어져요`
- 설명: `저장한 산책이 영역과 다음 목표 계산에 바로 반영돼요.`
- 사용자 인식 목표: “산책 결과가 홈 목표와 연결된다.”

#### 카드 3. 시즌
- 제목: `자주 걷는 칸일수록 시즌 점령이 진해져요`
- 설명: `같은 구역을 꾸준히 걸으면 시즌 지도에 더 강하게 남아요.`
- 사용자 인식 목표: “시즌은 별도 게임이 아니라 산책 기록의 후속 결과다.”

#### 카드 4. 미션
- 제목: `미션은 산책 위에 얹히는 보조 흐름이에요`
- 설명: `기본은 산책 기록이고, 미션은 그 위에서 추가 보상을 주는 보조 시스템이에요.`
- 사용자 인식 목표: “미션이 기본 루프를 가리지 않는다.”

## CTA 구조
- 주 CTA: `이해했어요`
  - 다음 단계: Step2 기록 방식/공유 기본값 선택
- 보조 CTA: `나중에 다시 보기`
  - 동작: 온보딩을 닫고 지도 첫 진입 가이드에서 다시 열 수 있게 함

## surface 연결 규칙
- Step1은 `첫 산책 전용 집중 설명`이다.
- 상시 화면에는 긴 설명 카드를 남기지 않는다.
- 상시 재열기 경로는 아래 두 곳으로 제한한다.
  - 지도 첫 진입 가이드 sheet
  - 시작 전 compact helper의 `가이드 다시 보기`

## 후속 UI 구현 힌트
- 표현 밀도는 `페이지형 풀스크린 1장 + 카드 4장 + CTA 2개`를 기준으로 한다.
- 페이지 수를 늘리는 대신 한 화면에서 스크롤 길이를 짧게 유지한다.
- Step1을 다 본 뒤에만 Step2로 넘어가게 하되, 강한 차단 대신 `다음` 흐름으로 연결한다.

## 금지 규칙
- 첫 화면부터 기록 방식/공유 기본값 버튼을 보여주지 않는다.
- 실내 미션을 첫 카드나 가장 큰 시각 요소로 올리지 않는다.
- 영역/시즌/미션을 한 문단으로 뭉쳐 설명하지 않는다.

## 구현 완료 판단 기준
- 사용자가 `기록 -> 영역 -> 시즌 -> 미션` 순서를 혼동하지 않는다.
- 지도/홈에 장문 카드가 상시 남지 않는다.
- Step2가 설정 선택 단계로 자연스럽게 이어진다.
