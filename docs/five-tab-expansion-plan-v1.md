# Five-Tab Expansion Plan v1.1

## 배경
- 현재 IA는 `홈 / 산책 목록 / 지도 / 설정` 4탭 구조.
- 기존 중앙 강조 탭은 5탭 전제에서 설계된 시각 패턴이었고, 4탭 전환 이후 정보 구조와 불일치가 발생.
- 동시에 시즌/퀘스트/라이벌 기능이 홈으로 밀집되어 탐색 비용이 커짐.

## 목표
- IA를 5탭으로 확장해 기능 역할을 명확히 분리한다.
- 캐리커처는 전용 탭이 아니라 프로필 맥락에서 유지한다.

## 권장 5탭 안 (사용자 제안 반영)
1. 홈
2. 산책 목록
3. 지도 (중앙 강조)
4. 라이벌
5. 설정

### 라이벌 탭 역할
- 근처 사용자 익명 핫스팟
- 라이벌 매칭/리더보드
- 신고/차단/프라이버시 동의 제어
- 향후 친구 비교 확장 진입점

## 왜 라이벌 탭이 맞는가
- 현재 열린 이슈와 직접 정렬: #130, #131, #132
- 홈 화면에서 경쟁/사회적 문맥을 분리해 정보 과밀 완화
- 사용자 행동 동선 명확화: 기록(지도) vs 경쟁(라이벌)

## 단계별 실행
### Phase 1
- 5탭 IA 확정 + 라이벌 탭 skeleton
- 기본 콘텐츠: 익명 핫스팟 카드, 동의/비동의 상태, 빈상태 UX

### Phase 2
- 라이벌 매칭/리더보드 연결
- 신고/차단/프라이버시 UX 연동

### Phase 3
- 친구 기반 확장(선택)
- 시즌/퀘스트 보상과 라이벌 성과 연결

## UX 원칙
1. 라이벌 탭은 익명/동의 기반이 기본값.
2. 개인 식별 정보(닉네임/강아지명/정밀 좌표) 비노출 원칙 유지.
3. 캐리커처는 프로필 편집/생성에서만 제공.

## 수용 기준
- 5탭 구조에서 각 탭 책임이 명확히 분리된다.
- 홈 과밀도가 완화되고, 경쟁 기능 진입점이 일관화된다.
- 프라이버시 기본값(비공개) 정책이 탭 UX에서 명확히 노출된다.
