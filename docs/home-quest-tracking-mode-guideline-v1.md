# Home Quest Tracking Mode Guideline v1

## 목적
- 홈 미션 카드에서 `자동 기록`과 `직접 체크`를 첫 표면에서 바로 구분하게 한다.
- 홈/지도/위젯으로 같은 개념이 확장돼도 용어와 행동 기대가 어긋나지 않게 한다.

## Canonical Naming
- 자동형 badge: `자동 기록`
- 자동형 title: `산책 중 자동 반영`
- 자동형 subtitle: `산책 데이터로 진행돼요`
- 직접형 badge: `직접 체크`
- 직접형 title: `실내 행동 직접 기록`
- 직접형 subtitle: `끝낸 행동만 직접 남겨요`

## 금지 용어
- `manual`
- `fallback`
- `finalize`
- `claimable`

## 카드 표면 규칙
- 추적 방식은 badge + icon + copy 중 최소 2개 이상으로 중복 전달한다.
- 버튼보다 먼저 추적 방식을 설명한다.
- 사용자가 규칙을 추론하게 두지 않는다.

## 자동 기록형 규칙
- 어떤 데이터로 진행되는지 바로 보여준다.
  - 예: 산책 시간, 이동 거리, 영역 변화
- 앱을 계속 열어두지 않아도 저장된 산책 데이터 기준으로 반영된다는 점을 설명한다.
- 조건 충족 시 상태가 자동으로 바뀐다는 점을 밝힌다.

## 직접 체크형 규칙
- 실제로 끝낸 행동만 기록해야 한다는 점을 먼저 말한다.
- 필요한 횟수를 카드에서 즉시 보여준다.
- 횟수를 채운 뒤에도 `완료 확인` 또는 `보상 받기`로 한 번 더 확정해야 한다는 점을 설명한다.
- `행동 +1 기록` 버튼만 보고 의미를 추측하게 하지 않는다.

## 재사용 규칙
- Home: 비교 카드 + 행 단위 badge/요약 문구를 함께 사용한다.
- Map: 자동형 설명만 노출할 때도 `자동 기록` / `산책 중 자동 반영` 조합을 유지한다.
- Widget: 짧은 표면에서도 `자동 기록` / `직접 체크` 용어를 그대로 유지하고 축약된 subtitle만 사용한다.

## 검증 포인트
- 카드 한 장만 보고 자동형/직접형을 구분할 수 있어야 한다.
- 직접 체크형에서 왜 `행동 +1 기록`과 `완료 확인`이 둘 다 필요한지 이해돼야 한다.
- 자동형을 직접 체크형으로, 직접형을 자동형으로 오해하지 않아야 한다.
