# Weather UX/Fallback/Accessibility v1 (Issue #135)

## 목표
- 날씨 기반 목표 치환과 스트릭 보호 상태를 한 화면에서 이해 가능하게 만든다.
- 날씨 데이터 실패 시 기본 퀘스트 유지 + 명확한 fallback 안내를 제공한다.
- 색상 의존을 줄이고 텍스트/아이콘/접근성 라벨로 상태를 함께 전달한다.

## 홈 UX 구성
1. 날씨 상태 배지 + 오늘 목표 변경 카드
- 배지: `정상`, `치환`, `Fallback`
- 카드 정보:
  - 치환 사유
  - 적용 시점(오늘 HH:mm)
  - 보호(Weather Shield) 사용량

2. fallback 동작
- 날씨 리스크 소스가 없거나 장애 상태면 `fallback`으로 판정
- fallback 시 리스크를 `clear`로 취급해 기본 퀘스트를 유지
- 사용자 문구: "날씨 데이터 연결이 불안정해 기본 퀘스트로 운영 중"

3. 스트릭 보호 결과 카드
- 보호 적용이 발생한 날은 당일 요약 카드 1종을 노출
- 내용: 오늘 보호 적용 횟수 + 마지막 적용 시각

## 지도 UX 구성
- 상단에 날씨 상태 칩 노출
- fallback 상태일 때 `Fallback` 텍스트와 안내 색상 표시

## 접근성/다국어
- 상태 전달은 색상 + 텍스트 + 아이콘 병행
- 접근성 라벨에 "리스크/치환 사유/적용 시점/보호 횟수"를 한 문장으로 제공
- 기본 문구는 한국어, 영어 기기에서는 핵심 상태 문구를 영어로 fallback 제공

## 수용 기준 매핑
- 치환 이유 이해: 홈 카드에 `치환 사유/적용 시점` 노출
- API 장애 UX 유지: fallback 배지 + 기본 퀘스트 유지
- 보호 결과 전달: 당일 요약 카드로 횟수/시점 제공

## 검증 포인트
- 위험 날씨: 치환 카드 + 보호 사용량 노출
- 데이터 실패: fallback 배지 + 기본 퀘스트 유지
- 접근성: 카드 accessibility label이 상태를 완결 문장으로 읽음
