# Settings Privacy Center IA v1

- Issue: #700
- Relates to: #268, #266

## 목표
- 설정 탭 안에서 사용자가 `프라이버시 관련 제어는 여기서 한다`고 바로 이해할 수 있는 진입 구조를 고정한다.
- 실시간 공유를 쓰지 않는 사용자도 현재 상태를 읽고 다음 행동을 결정할 수 있어야 한다.

## 핵심 원칙
1. 프라이버시 제어는 한 허브에 모은다.
2. 공유 기능 사용자만 이해할 수 있는 내부 용어를 쓰지 않는다.
3. 즉시 제어와 문서/안내를 같은 화면에서 분리해 보여준다.
4. 지도/라이벌의 빠른 토글은 shortcut이고, canonical source는 설정 탭의 프라이버시 센터다.

## 설정 탭 IA에서의 위치

### 상위 레벨
- `프로필`
- `반려견 관리`
- `시즌/앱 설정`
- `프라이버시 센터`
- `문서/지원`
- `계정`

### 진입 위치
- 설정 메인 화면의 독립 섹션으로 둔다.
- 제목은 `프라이버시 센터`
- 보조 문구는 `공유 상태, 권한, 보존/삭제 요청을 한 곳에서 관리해요`

### 진입 버튼 규칙
- 현재 공유 상태를 badge로 함께 보여준다.
  - `비공개`
  - `공유 중`
  - `권한 필요`
- badge만으로 의미를 전달하지 않고 텍스트 subtitle도 함께 둔다.

## 허브 화면 구조

### 헤더
- 제목: `프라이버시 센터`
- 보조 문구: `공유 상태와 권한, 보존/삭제 요청을 여기서 확인할 수 있어요`

### 카드 순서
1. 현재 공유 상태 카드
2. 즉시 비공개/재개 카드
3. 권한 상태 카드
4. 최근 공유 상태 카드
5. 삭제 요청 카드
6. 보존/삭제/문서 카드

이 순서를 유지한다. 문서 카드를 맨 위로 올리지 않는다.

## 카드 상세 정의

### 카드 1. 현재 공유 상태
- 목적
  - 지금 이 계정/기기에서 공유가 켜져 있는지 한 줄로 이해시킨다.
- 표시 항목
  - 현재 상태: `비공개` / `공유 중`
  - 마지막 반영 시각
  - 상태 설명 1줄
- 상태 예시
  - `지금은 비공개예요. 산책 기록은 남지만 근처 공유에는 참여하지 않아요.`
  - `현재 익명 공유 중이에요. 산책 중에는 근처 열기 집계에 반영될 수 있어요.`

### 카드 2. 즉시 비공개/재개
- 목적
  - 사용자가 가장 자주 찾는 제어를 한 번에 처리한다.
- 액션
  - `지금 바로 비공개`
  - `다시 공유 시작`
- 규칙
  - 세션 단위 즉시 비공개와 전역 공유 기본값 변경은 구분해서 설명한다.
  - 허브 첫 화면에서는 `즉시 현재 상태를 끄는 행동`을 우선 배치한다.

### 카드 3. 권한 상태
- 목적
  - 위치/알림 권한이 실제 공유 경험에 어떤 영향을 주는지 보여준다.
- 표시 항목
  - 위치 권한
  - 알림 권한
  - 복구 버튼 `설정 열기`
- 사용자 문구
  - `위치 권한이 꺼져 있어 근처 공유를 다시 시작할 수 없어요`
  - `알림이 꺼져 있어 공유 상태 변경 안내를 놓칠 수 있어요`

### 카드 4. 최근 공유 상태
- 목적
  - 실시간 공유를 쓰지 않는 사용자도 `최근에 어떤 상태였는지` 이해하게 만든다.
- 표시 항목
  - 마지막 공유 시각
  - 마지막 공유 결과
  - 최근 오류/보류 여부
- 빈 상태 문구
  - `아직 공유한 기록이 없어요`

### 카드 5. 삭제 요청
- 목적
  - 일반 문의와 분리된 `접수 가능한 삭제 요청 흐름`을 제공한다.
- 표시 항목
  - 현재 삭제 요청 상태
    - `접수 전`
    - `초안 준비됨`
    - `전송 확인 대기`
    - `회신 대기`
  - 요청 ID
  - 다음 단계 설명
- 액션
  - `삭제 요청 흐름 열기`
  - `처리 상태 문의 열기`
- 규칙
  - request id를 먼저 발급한다.
  - Mail composer 우선, `mailto:`는 fallback으로만 둔다.
  - 일반 지원 문의와 제목/배지/단계를 섞지 않는다.

### 카드 6. 보존/삭제/문서
- 목적
  - 사용자가 정책/보존기간/삭제 요청으로 자연스럽게 넘어가게 만든다.
- 포함 항목
  - `보존 기간 안내`
  - `삭제 요청 처리 기준`
  - `개인정보처리방침`
  - `이용약관`

## 게스트 / 회원 차이

### 게스트
- 허브는 읽기 전용 안내 중심
- 보여줄 것
  - 현재 상태: `로그인 후 공유 기능 사용 가능`
  - 문서/보존 정책
  - 로그인 진입 CTA
- 숨길 것
  - 서버 반영 전제의 최근 공유 이력

### 회원
- 전체 카드 표시
- 상태/권한/최근 이력/삭제 요청까지 모두 접근 가능

## shortcut vs canonical route

### shortcut surface
- 지도 설정 시트의 `위치 공유`
- 라이벌 탭의 `익명 공유 시작/끄기`

### canonical route
- `설정 탭 > 프라이버시 센터`

규칙:
- shortcut에서 상태를 바꿔도, 상세 설명과 장기 제어는 프라이버시 센터에서 보게 한다.
- 분산 제어가 아니라 `빠른 제어 + 허브 세부 관리` 구조를 유지한다.

## 용어 표준
- `공유 중`
- `비공개`
- `지금 바로 비공개`
- `다시 공유 시작`
- `권한 필요`
- `최근 공유 상태`
- `보존 기간`
- `삭제 요청`

쓰지 않을 용어:
- `visibility`
- `presence`
- `opt-in`
- `suppression`

## 후속 구현 메모
- 설정 메인에 `프라이버시 센터` 섹션 추가
- Settings action card를 재사용하되 privacy 전용 status/badge를 붙일 수 있어야 함
- 권한 상태와 최근 상태는 `summary card`로, 문서는 `action row`로 분리

## 완료 판단 기준
- 설정 탭에서 프라이버시 제어의 위치가 명확하다.
- 실시간 공유를 안 쓰는 사용자도 허브에서 자기 상태를 이해할 수 있다.
- shortcut과 canonical route의 역할이 충돌하지 않는다.
