# Planning Finalization Pack v1 (2026-03-04)

## 1) 목적
- `#230~#242` 이슈를 `HOLD` 상태에서 확정 가능한 상태로 끌어올리기 위한 최종 의사결정 패키지다.

- 확정 전에는 `status:hold`를 유지한다.

## 2) 확정 원칙
1. 프라이버시 우선: 실시간 위치 기능은 기본값 OFF + 명시적 동의(opt-in) 원칙.
2. 성능 안전: 지도/배터리/네트워크 비용은 기능 가치보다 우선 검증.
3. 운영 가능성: 관측 지표/롤백/킬스위치 없이 출시하지 않음.
4. 단계적 확정: 저위험 UX부터 확정 -> 고위험 실시간 공유는 정책 게이트 통과 후 확정.

## 6) 권장 확정값 v1 (수치 잠금 제안)
1. #230 route 반복 감쇠 수식
- 식: `route_bonus_ratio = 0.20 * novelty_ratio * repeat_decay`
- `novelty_ratio = unique_route_cells / max(1, total_route_cells)`
- `repeat_decay = max(0.40, 1.0 - 0.12 * max(0, repeat_pass_count - 1))`
- 최종: `route_bonus_ratio <= 0.20` 강제

2. #233 알림 상한/큐 우선순위
- 일일 로컬 알림 상한: `2회` (인앱 팝업은 상한 없음, 단 최소 간격 적용)
- 인앱 최소 간격: `20초`
- 우선순위: `권한/복구/자동종료 경고 > 시즌결과/퀘스트완료 > 명소배지`

3. #237 TTL/SLA
- TTL: `90초`
- 조회 SLA: viewport 조회 `p95 <= 350ms`, `p99 <= 700ms`
- 업서트 SLA: `p95 <= 250ms`

4. #239 marker cap (기기급 기본값)
- Low tier: `80`
- Mid tier: `120`
- High tier: `180`
- 공통: 저배율에서는 클러스터 기준 cap(`<= 60` cluster) 우선 적용

5. #242 단계 승급 KPI 임계값
- 공통 게이트: `error_rate < 1.0%`, `stale_ratio < 12%`, `p95_latency < 350ms`
- 배터리: 30분 산책 기준 기능 OFF 대비 추가 소모 `<= +5%p`
- 단계 승급: 10%(`24h`), 50%(`48h`), 100%(`72h`) 연속 충족 시만 허용
