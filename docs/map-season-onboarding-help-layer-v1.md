# Map/Home Season Onboarding Help Layer v1

## 결정
- 최초 설명 위치는 `지도 시즌 요약`으로 고정한다.
- 이유:
  - 사용자가 실제 `타일`, `점령`, `유지`를 보면서 바로 이해할 수 있다.
  - 홈 시즌 카드만으로는 `칸`과 `테두리`의 의미를 설명하기 어렵다.
- 재진입 위치는 `지도 시즌 요약`과 `홈 시즌 카드` 두 곳으로 둔다.

## 최초 노출 정책
- `실사용자` 기준
  - 시즌 타일이 실제로 보이는 첫 지도 진입에서 1회 자동 노출
- `UI 테스트` 기준
  - 기본 자동 노출 비활성
  - `-UITest.SeasonGuideAutoPresent`일 때만 강제 노출

## 사용자 문구 기준
- 내부 용어 노출 금지
  - `effective_score`, `tile_events`, `anti-farming`
- 사용자 행동 중심 문장 사용
  - `새로운 칸을 넓히면 더 큰 기여가 반영돼요`
  - `이미 넓힌 칸을 다시 걸으면 유지 기여가 쌓여요`
  - `같은 자리만 짧게 반복하면 점수가 거의 늘지 않을 수 있어요`

## 시트 구성
1. 헤더
   - `처음 보는 시즌` / `지도에서 다시 보기` / `홈에서 다시 보기`
2. 핵심 개념 카드
   - 시즌 타일
   - 점령
   - 유지
   - 새 타일 가치
3. 반복 경로 규칙
4. 산책 -> 시즌 결과 5단계 흐름
5. 다시 찾는 방법 안내

## 재진입 CTA
- 지도: `시즌이 뭔가요?`
- 홈: `시즌이 뭔가요?`

## 상태 저장
- `season.guide.initial.presented.v1`
- 자동 노출 소비 여부만 저장
- 수동으로 가이드를 연 경우도 `seen`으로 처리해 이후 자동 노출을 막는다

## QA 포인트
- 첫 지도 진입에서 가이드가 자동으로 1회만 뜨는가
- 홈 시즌 카드에서 가이드를 다시 열 수 있는가
- 지도 시즌 요약에서도 가이드를 다시 열 수 있는가
- 설명 문장이 `타일`, `점령`, `유지`, `새 타일 가치`, `반복 경로 규칙`을 모두 다루는가
