# Rival Stage3 Client UX v1

## 목표
- 라이벌 비교 UI를 스켈레톤에서 실제 동작 가능한 상태로 전환한다.
- 신고/차단/숨기기 액션을 2탭 이내로 접근 가능하게 제공한다.
- opt-out(익명 공유 OFF) 시 비교 데이터를 즉시 비활성화한다.

## 화면 정책
- 진입 배지:
  - `공유 중/비공개`
  - `위치 허용/권한 필요/로그인 필요`
- 비교 범위:
  - `라이벌` (실데이터)
  - `친구` (프리뷰 상태)
- 비교 기간:
  - `주간`
  - `시즌`

## 데이터/노출 가드
- 리더보드는 `rpc_get_rival_leaderboard` 결과를 사용한다.
- 점수는 원문 숫자 대신 `score_bucket`만 노출한다.
- 정밀 좌표/닉네임/강아지명은 노출하지 않는다.
- 핫스팟은 geohash7 기반 집계 요약만 표시한다.

## 신고/차단/숨기기 UX
- 각 리더보드 행 우측 메뉴에서 즉시 실행:
  - 신고
  - 차단
  - 숨기기
- 신고 사유:
  - 부적절한 활동
  - 스팸/도배
  - 기록 조작 의심
- 차단/숨기기 대상은 즉시 목록에서 제거한다.
- 숨김/차단 관리 시트에서 해제 가능하다.

## 상태 전이
1. 비로그인: 비교 잠금 상태
2. 로그인 + 권한 미허용: 권한 안내
3. 로그인 + 권한 허용 + 공유 OFF: 동의 안내
4. 로그인 + 권한 허용 + 공유 ON: 핫스팟/리더보드 조회
5. 공유 OFF: 핫스팟/리더보드 즉시 비활성

## 로컬 저장 키
- `rival.hidden.alias.codes.v1`
- `rival.blocked.alias.codes.v1`
- `rival.moderation.logs.v1`

## 수용 기준
- 동의 전/후 UI 분기가 즉시 반영된다.
- 신고/차단/숨기기 이후 목록이 즉시 갱신된다.
- opt-out 시 비교 UI가 즉시 비활성화된다.
- 비교 카드에서 정밀 위치 유추 정보가 표시되지 않는다.
